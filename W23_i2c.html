MiniLab: An I2C Sensor

In this short exercise, you will learn how to interface a microcontroller with an I2C sensor. This will be very important to understand soon when you will use a more complicated I2C interfaced sensor.

Background

Many useful sensors communicate with a microcontroller using a digital interface such as an SPI or I2C bus. The I2C interface is especially popular because its addressing scheme allows many sensors to be connected to a microcontroller using only two GPIO pins plus power and ground. A typical sensor is the MMA8451Q accelerometer from NXP Semiconductors. This three-axis accelerometer can be configured to measure accelerations of ±2g, ±4g, or ±8g at rates of over 500 measurements per second.

Any sensor should be calibrated. MEMS accelerometers are easy to calibrate at ±1g using gravity; just stand on Earth, not too far from sea level, and hold the accelerometer still in a position such that each axis points up and then down to see ±1g on that axis.

The Project

  • Download mma845x_shell.py from the ME405-Support repository examples folder into a new directory such as I2C_lab for this project. Rename the file mma845x.py.

  • With USB power off as usual, connect your sensor to the Shoe of Brian screw terminal connectors as shown below.

    The Shoe of Brian has two pullup resistors RPU of about 4.7KΩ already installed. If not using a Shoe, you may need to put in two 4.7KΩ ~ 10KΩ resistors as shown. Leave other lines such as I1, I2, etc. on the sensor board disconnected.

  • Look up the data sheet for the MMA8451Q accelerometer. If you have an MMA8452Q instead, not to worry; it’s the same data sheet. Skim through the first page.

  • Make sure the I2C system is working and the accelerometer is talking by running the following in the REPL:

      aye = pyb.I2C (1, pyb.I2C.CONTROLLER)
  pyb.I2C.scan (aye)

    The call to scan() should return a list containing at least one decimal (not hex!) number, the I2C bus address of the accelerometer (see Table 10 on data sheet page 17).

  • Fix the code in mma845x.py so that the method which returns X acceleration works rather than displaying a whiny message that it can’t. Note that you should fix the method that returns binary numbers first, then test it, then fix the method that returns calibrated acceleration. Read the hints below.

  • Grab a fresh copy of basic_tasks.py from the ME405-Support GitHub site. Modify it so that task1_fun() reads the accelerometer’s X acceleration two times per second and puts each reading into a queue, and task2_fun() gets each reading from the queue and prints it.

  • Verify the calibration of the accelerometer by the tilt test described above. Make a note of the calibration accuracy that you have measured.
  • Before the end of the lab period, demonstrate your working accelerometer program for your instructor.

Hints

  • Notice that active() and standby() work in the provided code and can be used as examples of how to read and write registers in the accelerometer.

  • It’s easiest to read acceleration by using the i2c.mem_read() function to read two bytes from register addresses beginning at the accelerometer’s X axis MSB register; this returns a bytearray containing the MSB and LSB. Use int.from_bytes() to convert the two-byte array to one integer.

  • You get acceleration as a 16-bit twos-complement number in an int, so you need to correct negative numbers so that they are not huge and positive. Just subtracting 216 from such numbers works.

Deliverables

  • Put your accelerometer driver file mma845x.py and modified basic_tasks.py into a .zip archive and submit them to the I2C Mini-Lab assignment on Cantvas.
